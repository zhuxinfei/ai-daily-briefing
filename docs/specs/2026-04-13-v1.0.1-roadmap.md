## 高优先级详细设计

### H1 — 站点主页 Landing Page

**现状**: 访问 `/` 显示 Hextra Hero 默认骨架 + 空 cards  
**目标**: 完整的 Landing Page，首次访客一眼看懂"这是什么 + 怎么用"

**内容结构**:
1. **Hero 区**: AI 日报品牌标语 + "每天 8:30 全网 AI 资讯聚合"
2. **最近 6 期卡片**: 自动从 content/cn/ 读最新 6 篇，展示日期 + 摘要
3. **特色介绍**: 3 列卡片（全自治 / 多维筛选 / 在线对话）
4. **"查看今日早报"按钮**: 动态指向最新一期

**技术方案**:
- 改 `content/cn/_index.md` (Hextra Hero + cards shortcode)
- 在 `hugo.go` 的 WriteHugoPost 末尾加 `updateHomepage()` 函数
  - 自动读 content/cn/ 最新 6 篇 md 的 frontmatter (title + date + description)
  - 更新 `_index.md` 的 LATEST_6_CARDS 占位区
  - 更新"查看今日早报"按钮链接
- 不动 hugo.yaml / theme / Slack / PIL

**影响评估**: 加 1 个新函数到 hugo.go，不影响现有 WriteHugoPost / render / publish

**参考**: ai.hubtoday.app 首页（独立 Landing + 侧边栏月份档案）

### H2 — 导航修复

**现状**: "AI资讯日报" 指向空首页，"AI周报" 指向空页面  
**目标**: 两个导航都可用

**方案**:
- **AI资讯日报** (pageRef: `/`): H1 做完后首页有内容，导航自然可用
- **AI周报** (pageRef: `/blog/weekly`): 保留导航，页面显示"AI 周报即将上线"占位（不是空白）
  - 改 `content/cn/blog/weekly/_index.md` 加占位内容
  - 实际周报 pipeline 放 v1.0.2

**影响评估**: 改 2 个 md 文件，不动代码

### H3 — gate warn 时自动 retry

**现状**: gate warn → 推 test + skip prod → service exit 1 → 用户收到"质量待审"  
**目标**: gate warn → 不推任何频道 → 30 分钟后自动重跑 → 直到 pass 或重试 3 次

**方案**:
- 新建 `briefing-daily-retry.timer` (OnActiveSec=30min, 一次性)
- `briefing-daily.service` 的 OnFailure 加 `briefing-daily-retry.timer`
- gate warn 时 service exit 1 → systemd 触发 retry timer → 30min 后再跑
- 加 `RETRY_COUNT` 环境变量，最多重试 3 次，超过就 alert + 推 test (让 developer 看到)

**影响评估**: 新建 1 个 timer + 改 service OnFailure，不动 Go 代码

## 中优先级详细设计

### M1 — infocard LLM prompt 升级

**现状**: LLM prompt 还是旧 schema (6 stories / 3 numbers / 单行 sub_headline)，靠 enrichLLMHeader 补  
**目标**: LLM 直接输出 14 stories / 6 numbers / `\n` 分隔的 multi-line sub_headline

**方案**: 改 `internal/infocard/infocard.go` 的 `infoCardSystemPrompt`，更新 JSON schema 要求
**影响**: 只改 prompt 字符串，不动渲染逻辑

### M2 — LLM 洞察/启发 prompt 不硬凑

**现状**: prompt 可能仍写"输出 4 条行业洞察"，跟"不硬凑"原则矛盾  
**目标**: prompt 改成"根据今日内容客观分析，有多少输出多少"

**方案**: 改 `internal/generate/` 的 insight prompt
**影响**: 只改 prompt 字符串

### M3 — title-based dedup

**现状**: dedup 只按 URL 完全匹配，不同源同话题会重复  
**目标**: 加 title 关键词匹配（提取 4+ 字符英文专有名词作为 dedup key）

**方案**: 在 `dedupRawItemsBySent` 加 title keyword 维度，`sent_titles.txt` 持久化  
**影响**: 加文件 + 改 dedup 函数，不影响其他 pipeline 阶段

### M4 — cloudflared Named Tunnel

**现状**: Quick Tunnel 每次重启换 hostname，靠 sync 脚本补  
**目标**: 绑定固定 subdomain（e.g. daily.ominieye.dev）

**方案**: 需要用户 cloudflared login → tunnel create → DNS route → 改 systemd  
**影响**: 改 systemd service + secrets.env，不动 Go 代码  
**前提**: 需要 Cloudflare 账号 + 域名

### M5 — Mac 本地 GitHub SSH key

**现状**: Mac 的 SSH key 关联 xinfeizhu (suspended)，git pull 失败  
**目标**: Mac 配新 SSH key 关联 ylzsdafei

**方案**: `ssh mac "ssh-keygen -t ed25519"` → 用户加 pub key 到 GitHub  
**影响**: 只改 Mac SSH 配置
