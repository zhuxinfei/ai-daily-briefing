# briefing-v3 v1.0.0 Hextra 迁移方案

本文件是 v1.0.0 升级改造的 single source of truth。Agent Team 的每个 teammate 在开始工作前都必须通读一遍。后续所有 checklist 项、验收标准、文件边界、协作规则都以本文件为准。本文件允许在实施过程中补丁修订，但重大架构决策变更必须同步回写这里。

---

## 1. 背景与一句话目标

现有 briefing-v3 (v1.0.0-rc5) 已经具备 21 源并发采集、SQLite 永久存储、LLM 打分分类撰稿、Slack 双通道推送、promote snapshot 等能力，但前端是自写 HTML（internal/render/html.go 1434 行 + docs/\*.html），与上游 ai.hubtoday.app 的 Hugo + Hextra 架构完全脱节；同时 classify 严重偏斜和 compose 502 硬失败两个 bug 导致 2026-04-10 起连续两天早报凑不齐 20 条、5 section 有空洞、gate fail。

**v1.0.0 一句话目标**：用 justlovemaki/Hextra-AI-Insight-Daily 作为前端基座，把 briefing-v3 现有的 markdown 渲染和 Slack 推送接上去，同时修掉 classify 偏斜和 compose 502，一比一复刻 ai.hubtoday.app 的前端外观与信息架构，完全自治，每天 Asia/Shanghai 08:30 自动触发并推送到 Slack 测试频道。

## 2. v1.0.0 作用域

### 必做（对标 ai.hubtoday.app 和用户显式需求）

- Hugo + Hextra 静态站点（仅中文）
- 21 源 ingest → rank → classify → compose → insight → 6 段 markdown 的完整 pipeline

- SQLite 永久存储（issues / raw_items / issue_items / issue_insights / deliveries 五张表已就位）
- SQLite 每日备份快照
- 修 classify 偏斜（research 占比降到 35%-40%）
- 修 compose 502 硬失败（单 section 降级 + pipeline 继续）
- Gate 在 test 频道支持 warning 降级推送（不静默失败）
- 失败告警到 Slack test

### 不做（留给 v1.0.1+）

- 导航站 nav.hubtoday.app
- 飞书云文档和飞书群 bot
- 周报
- 英文版 content/en/
- L1 头图 PNG（internal/image, scripts/gen_headline.py）
- L2 信息卡 PNG（internal/infocard, scripts/gen_info_card.py）
- Pollinations 插图
- Named Cloudflare Tunnel（继续用 Quick Tunnel）
- Gemini 模型接入
- wire_api = responses 切换
- 跨日 URL 去重
- Giscus 评论
- Google Analytics / AdSense / Umami

## 4. 架构决策表

| 维度 | 选择 | 理由 |
|---|---|---|
| 前端基座 | clone `justlovemaki/Hextra-AI-Insight-Daily` 到 `/root/ai-daily-site/` | 真正站上肩膀，不重造主题 |
| 前后端关系 | 两个独立目录，不做 git submodule | briefing-v3 通过 `HEXTRA_SITE_DIR` 环境变量定位前端目录 |
| markdown 源 | 复用 `internal/render/markdown.go`（已是 6 段布局），新增 `internal/render/hugo.go` 只做 frontmatter 包装 | 最小改动，风险低 |
| 旧 html 路径 | 废弃但保留代码，在 run.go 里注释调用 | 可快速回滚 |
| classify 可靠性 | 先用 `sources.category` 规则映射到 section，LLM 只做边缘 item 的二次判断 | 规则优先于 LLM 判断，避免偏斜 |
| rank 均衡 | per-category quota（在 config/ai.yaml 增加字段） | 从源头保证 5 section 都有料 |
| compose 降级 | for 循环里 `summarize` 失败 log + continue，收集 `failedSections` 列表 | test 频道总能收到东西 |
| gate 策略 | test 频道 fail 降级为 warn 继续推送（消息头加"质量待审"），prod 频道保持硬失败 | 白噪声优于静默失败 |
| LLM API 路径 | 保持 `/v1/chat/completions`（04-10 已验证能跑） | 不引入 wire_api=responses 切换风险 |
| 模型 | 继续 gpt-5.4 | 不卡在 Gemini API key 申请 |
| 图片策略 | v1.0.0 只用 og:image（mediaextract），所有 PIL / infocard / Pollinations 路径跳过 | 砍复杂度 |
| giscus | hugo.yaml 里 `enable: false` 或注释 | 不卡在 repoId 申请 |
| 导航外链 | menu.main 删 `ai_nav`、`github/同性交友`两项 | 上游品牌移除 |
| 双语 | `defaultContentLanguage: zh`，删除或置空 `languages.en` | MVP 先中文 |
| HTTP 服务 | 新建 `briefing-site.service` listen :8081 serve `/root/ai-daily-site/public`，保留 `briefing-serve.service` :8080 作为回滚路径 | 避免 restart briefing-serve 导致 tunnel 漂 |
| Tunnel | 保留 `cloudflared-tunnel.service`，改 `ExecStart` `--url http://localhost:8081`，restart **只** cloudflared 本身 | 影响最小 |
| 调度 | systemd timer (`*-*-* 08:30:00`, Asia/Shanghai) + OnFailure alert | journalctl 天然观测 |
| 铁规保护 | systemd timer service `ExecStart` 强制 `--target test`，prod 只能人工 `briefing promote --date YYYY-MM-DD` | 代码+配置双层保护 |
| 备份 | 每日 00:30 把 `data/briefing.db` 拷贝到 `data/backups/briefing-YYYY-MM-DD.db`，保留 30 天 | 防丢 |

## 7. 风险与回滚

| 风险 | 触发场景 | 应对 | 回滚点 |
|---|---|---|---|
| Hugo build 失败 | 主题 module 拉不下来 / yaml 改错 | run.go 里 build 非阻塞，只 log warn；Slack 照样推 | `git reset --hard` `/root/ai-daily-site` |
| classify 规则映射仍偏斜 | 某天全是 arxiv 论文 | per-category quota 兜底截取 + gate warn | 回滚 C1-C2 |
| compose 降级后 section 不足 5 | LLM 网关全挂 | gate warn + test 推送带"质量待审"标记 | 无需回滚，告警触发人工介入 |
| hugo.yaml 改错站点挂 | 误删关键字段 | 从 clone 初始 commit `git reset --hard` | git 可恢复 |
| Slack 误推 prod | timer 配置错 --target | service ExecStart 强制 --target test + promote 需要手工输日期 | 代码+配置双层保护 |
| Tunnel URL 漂移 | 误 restart briefing-serve | 只 restart cloudflared-tunnel 服务本身 | cf-tunnel.log 抓新 URL |
| SQLite 损坏 | 磁盘满 / 程序崩溃 | `data/backups/` 每日快照 | 恢复到前一天 |
| 21 源全挂 | GFW / CloudFlare 封禁 | gate fail → 推 alert → 人工介入 | 源自愈后下次 cron 恢复 |
| wire_api 不兼容 | A1 curl 返回 4xx | 切换到 /v1/responses，改两处 URL + request/response 结构 | 回滚到 /v1/chat/completions |
| Hextra 主题和用户 Hugo 版本不兼容 | 版本 gap | 锁定 Hextra 某个 tagged release 或 hugo 回退版本 | 切换 Hextra 分支 |
