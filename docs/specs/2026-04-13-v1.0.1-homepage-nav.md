## 1. hubtoday 参考 — 甄别结论

| hubtoday 功能 | 我们态度 | 理由 |
|---|---|---|
| **独立 Landing Page** (价值主张 + 推荐路径) | ✅ **照搬精神** | 给首次访客一个清晰的"这是什么 + 怎么用"入口 |

### 2.3 "查看今日早报" 动态链接

首页的 `[查看今日早报 →]` 按钮需要动态指向**最新一期**日报。

**方案 A (简单)**: 硬编码 `/2026/2026-04/` (用户点进去看 sidebar 选日期)  
**方案 B (优雅)**: `briefing run` 每次跑完后自动更新 `_index.md` 里的"今日早报"链接为 `/2026/2026-04/2026-04-13/`  

**推荐 B**: 在 `internal/render/hugo.go` 的 `WriteHugoPost` 末尾加一个 `updateHomepage` 函数，自动更新 `_index.md` 里的最新链接 + 最近 6 期 cards。

**影响评估**:
- 改 `hugo.go` (加 1 个新函数)，不影响现有 WriteHugoPost / scrubAndRelocateImages
- 改 `_index.md` 的特定占位符区域 (LATEST_6_CARDS_START...END)，不影响 Hero 布局
- `briefing run` 流程多 ~0.5 秒 (读 content 目录 + 更新 _index.md)

## 3. 不在 v1.0.1 范围（放 v1.0.2+）

| 功能 | 理由 |
|---|---|
| AI周报 | 需要新 LLM pipeline (周汇总 prompt + 新 template)，工作量大 |
| AI导航 | 跟早报定位不符 |
| 博客 | 没有人力维护 |
| 关于页 | 内部工具不需要 |
| Named Tunnel | 需要 Cloudflare 账号配置 |
| title-based dedup | 需要改 store 层 |
| infocard prompt 升级 | 需要改 LLM prompt + test |
